# Template Selection Index

Fast-scan registry for choosing a template. Find the row whose **Use when** and **Tags** match the script segment, then open that template's `GUIDANCE.md` for the full selection criteria, narration rules, and reveal sequence.

Each template's machine-readable selection metadata lives in the YAML frontmatter of its `GUIDANCE.md` (`category`, `tags`, `layout`, `narration`, `timing`). This index is the human/LLM-readable summary of that frontmatter.

## How selection works

1. **Match category + intent.** Scan this table for the segment's shape (comparison, process, list, single-point, definition, ...).
2. **Confirm against guidance.** Open the candidate's `GUIDANCE.md` and check the **Use it when** / **Do NOT use it when** sections.
3. **Check the narration shape.** Each guidance file states how narration must be structured for that template (e.g. YinYang requires a linear, side-complete comparison). If the script can't be phrased that way, pick a different template or reprocess the script per that file's **Reprocessing directive**.
4. **Fixed-role scenes.** Every lesson opens with a `LessonTitle` card (the lesson title, derived from the content). The **course identity** on that card (the `courseTitle` text + `courseIconUrl` icon, top-left) must be **identical across every lesson of the course**, reuse the exact same title string and icon each lesson; never re-derive or vary them. The **course outline / course-intro agenda** ("in this course we'll cover…") uses `BulletList6Pills` and appears **only in the first lesson** of a course; don't recreate the course-outline in later lessons.
5. **Case studies use a company LOGO, not an icon.** If a beat introduces a worked example about a specific company, choose `CaseStudyIntro` and give it a **fictional company logo** from `Logos/Fictional-Company-Logos/` (`Company-<Name>-light`); for a real named product use a `Logos/Software/` logo. Do NOT resolve the company to an `Icons/` glyph. General concepts still use icons as normal.
6. **Apply the variety rule.** Within a single lesson, use each template **at most once** so the lesson feels varied. There is no default template, do not lean on `YinYang2Points` (or any one template) just because it is prominent. When two beats fit the same template, keep the best fit there and route the other to the next-best template that also fits (every shape has alternatives: comparisons across `ComparativePoints2` / `SplitscreenPointsV1` / `YinYang2Points`; lists/points across `Topic1Subtopics6` / `BigPoints3V1` / `CirclePoints4` / `IconPointsV1` / etc.). A repeat within one lesson is allowed **only by genuine exception** (no other template fits that beat), and must be justified. Aim for zero repeats per lesson.
7. **Course-level variety (soft recommendation).** Across the lessons of ONE course, avoid leaning on the same content templates lesson after lesson. Keep a reference of what earlier lessons used (`python script-pipeline/course-templates.py <courseId> --exclude <thisLesson>`) and, when a beat has **multiple templates that fit equally well**, prefer one that is **unused or least-used so far in the course**. This is a recommendation, NOT a hard rule: never sacrifice good fit for variety, only use course history to break ties among equally-good options. Structural templates (`LessonTitle`, `LessonGoal`, `LessonSummary`, `CaseStudyIntro`, `BulletList6Pills`) are expected to recur and are exempt. Variety is **course-scoped and resets for a new course**.
